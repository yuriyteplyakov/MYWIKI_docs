# Основные сущности Service Desk

> Incident = сбой сервиса
> Request = запрос пользователя
> Problem = корневая причина
> Change = изменение системы

## архитектура системы

> frontend - Интерфейс пользователя (формы, списки заявок, дашборды)
> backend - Логика системы (создание заявки, назначение исполнителя, отправка уведомлений)
> database - база данных хранит (пользователей, заявки, логи, процессы)
> integrations - Связи с другими системами (LDAP, ERP, CRM, Monitoring)

**пример интеграции:** Service Desk → ERP

## Service Desk почти всегда интегрируется с:
> LDAP для авторизации пользователей
> Email
> Monitoring (Например система мониторинга Если сервер падает: автоматически создается инцидент)
> ERP
