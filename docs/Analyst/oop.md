
# Что такое ООП

> Объектно-ориентированное программирование — это способ написания программ, где всё строится вокруг объектов, а не действий.

## Основные понятия

**Класс:** Чертеж или шаблон. Например, чертеж автомобиля: у него есть цвет, скорость, способность ехать.

**Объект (Экземпляр):** Реальная вещь, созданная по чертежу. Например, конкретная красная машина со скоростью 100 км/ч.

**Свойства (Данные):** Характеристики объекта (цвет, модель, объем двигателя).

**Методы (Поведение):** Функции, которые объект может выполнять (завестись, затормозить, посигналить).


# Основные принципы ООП

**Инкапсуляция**

> Скрытие внутренней структуры объекта. Объекту "Водитель" не нужно знать, как именно работает двигатель, ему достаточно метода «завести».

**Наследование**

> возможность создавать новый класс на основе существующего. Можно создать класс «Грузовик», который наследует все свойства «Автомобиля», но добавляет «Грузоподъемность».

**Полиморфизм**

> Способность объектов с одинаковым интерфейсом вести себя по-разному. Например, у классов «Собака» и «Кошка» есть метод «издать звук», но одна гавкает, а другая мяукает.

**Абстракция**

> Выделение только важных характеристик объекта, скрывая лишние детали (например, для системы учета машин неважен цвет салона).

## Зачем это нужно?
> Повторное использование кода: Написав класс один раз, можно создавать тысячи объектов.
> Понимание: Программу легче читать, так как она состоит из логических сущностей (Пользователь, Заказ, Товар).
> Масштабируемость: Удобно работать над большими проектами, деля их на части (классы).
